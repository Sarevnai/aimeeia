1. Sumário Executivo

  • Paridade real > paridade aparente quando se controla pela assimetria epistêmica. §2 · §5
  • Aimee puxa em pipeline operacional (Kanban, Tickets, AI Lab, multi-agente, realtime+áudio); Lais puxa em Visitas (M-04) e Captação (M-08). §6 · O-01 · O-02
  • Drift Identity/Memory/Soul/Governance é o mais relevante — não existe como camadas no código. Q-001 · O-08
  • PII em ai_traces (30.7k rows) é risco estratégico #1. Q-008 · R-01 · O-03
  • Mocks de billing/metrics bloqueiam comercialização. Q-011 · O-05
  • Modelo de acesso por código único da Lais é arquiteturalmente primitivo — Aimee já supera. D.A-01 · §11
Postura recomendada: diferenciar em governança e operação multi-agente, parear em Visitas e Captação, rejeitar o modelo de acesso por código e a UI monolítica Bubble.

2. Nota Metodológica de Paridade Epistêmica

LAIS.v1.0 — black-box

a11y-tree + text-walker em 12 rotas autenticadas. Zero screenshots, zero código, zero prompts, zero schema, zero logs. Bubble.io sem aria-* em 5+ rotas (M-02, M-04, M-08, M-09, parcial M-07) → conteúdo inobservável. O que parece polido pode esconder débito equivalente.

AIMEE.v1.0 — white-box

Código completo, 64 migrations, 43 edge functions, 22 prompt builders, 45 tabelas, 30.7k traces, churn, drift arquitetural, 13 débitos catalogados. Viés pessimista por natureza — vemos as cicatrizes da Aimee, não as da Lais.

Regra de leitura: toda célula de SWOT/diferenciais carrega marcador verificada não-verificada assimétrica. Sem marcador, a célula é inválida. Afirmações não-verificáveis vão para §5 (Assimetrias), não para §4.

11. Rejeições Deliberadas

🚫 Acesso por código compartilhado

(LAIS.M-11) Fere A-04 e A-07. Aimee já supera via roles granulares (D.A-01).

🚫 Runtime UI no-code (Bubble.io)

(LAIS.I-011) Teto de manutenibilidade e observabilidade — LAIS.X-002 evidencia o custo.

🚫 Configuração do agente como cards soltos

Aimee já tem contrato explícito (ai_directives/modules/behavior). Aprofundar, não regredir.

🚫 Funil linear hardcoded de 3 níveis

Aimee tem Pipeline Kanban configurável (D.A-02). Não regredir.

13. Postura Estratégica Recomendada

Diferenciar em governança (roles, DNC, kill-switch, LGPD pós-O-03), operação multi-agente (A-05 federado + tickets + Kanban) e observabilidade (AI Lab + ai_traces maskados). Parear em Visitas (O-01, inadiável pelo cutover 07/05) e Captação (O-02, narrativa comercial). Rejeitar o modelo de acesso por código, a UI no-code como runtime e o funil linear.

A tese competitiva é "Aimee é uma Lais com chão estrutural — multi-agente verdadeiro, governança de primeira classe, CRM espelhado, multi-tenant com roles" — mas essa tese só se sustenta publicamente depois que O-03 (PII), O-04 (qualification) e O-05 (billing) estiverem fechados. Antes disso, o chão é concreto pintado.
